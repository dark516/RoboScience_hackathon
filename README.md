## 1. Заголовок и краткое описание проекта
Frob Trash Collector Robot с манипулятором
Проект представляет собой автономную роботизированную платформу на базе Frob Robot с манипулятором для захвата объектов. Робот реализует визуальную навигацию и сортировку предметов (мусора) по цветовым меткам корзин с использованием компьютерного зрения.
Проект разработан командой из 5 человек в рамках международного RoboScience Hackathon. На хакатоне удалось собрать прототип, реализовать ключевой алгоритм визуальной навигации и сортировки, но из-за интенсивного тестирования и поломок механических узлов команда заняла 2 место.

## 2. Установка и запуск проекта

Клонируйте репозиторий:
```bash
git clone https://github.com/dark516/Frob_robot
cd Frob_robot
```
Подключение к реальному роботу:
```
ssh robot@r1.frob-robot.online
```
(Замените r1 на ID вашей платформы.) Альтернативно, найдите IP:
```
sudo apt install arp-scan
sudo arp-scan --localnetЗатем:textssh robot@<IP-адрес>
```
Сборка на роботе:
Перейдите в директорию:
```
cd ros2/src/ros2_ws/
```
Соберите пакеты:
```
colcon build
source install/setup.bash
```
Запуск:
```ros2 launch frob_bringup bringup.launch.py
```
Проверка компонентов:
Управление движением: rqt → Plugins → Robot Tools → Robot Steering (топик /cmd_vel).
Визуализация одометрии:textros2 launch frob_description display_odom.launch.py
Камера и манипулятор запускаются через основной launch-файл.
Отладка: ros2 topic list.


3. Основной функционал проекта

Ручное и автономное перемещение с дифференциальным приводом.
Управление манипулятором для захвата объектов.
Визуальная навигация и сортировка мусора по цветовым меткам корзин (компьютерное зрение).
Связь с низкоуровневым контроллером Arduino (ros2_arduino_bridge).
Публикация одометрии и визуализация (frob_odometry).
Графическое управление через rqt.
Интеграция камеры для детекции объектов.
Веб-интерфейс для мониторинга и управления (web_interface).
3D-модели робота (models).

6. Архитектура и структура проекта

Структура папок:
assembly_guide/ — руководство по сборке робота и манипулятора.
installation/ — инструкции по установке.
models/ — 3D-модели робота и манипулятора (URDF/Xacro).
ros2/ — основной workspace с пакетами (bringup, odometry, arduino_bridge, vision nodes).
web_interface/ — подмодуль веб-интерфейса.

Архитектура: hardware (Arduino + манипулятор) → ROS2 nodes (vision, control, navigation) → visualization (rqt, RViz, web).

## 8. Финальный текст-заключение
Проект важен как практическая реализация образовательной робототехники для экологической задачи — автономной сортировки и сбора мусора с использованием манипулятора и визуальной навигации. Несмотря на механические поломки во время интенсивного тестирования, команда успешно собрала прототип и заняла 2 место на RoboScience Hackathon.
Проект отличается интеграцией манипулятора, алгоритмом сортировки по цвету и готовой ROS2-базой, что делает его лучше простых мобильных платформ без захвата.
Возможные улучшения:

Усиление механической конструкции для надёжности.
Расширенное компьютерное зрение (YOLO для детекции объектов, не только цвет).
Полная автономия с Navigation2 и SLAM.
Симуляция в Gazebo для безопасного тестирования.
